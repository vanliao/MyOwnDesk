# MyOwnDesk 需求分析

## 项目目标

实现一个自用的远程桌面工具，通过自建中继服务器在 3-5 台个人设备之间互相远程控制。

## 核心需求

### 功能需求

| 编号 | 需求 | 优先级 | 说明 |
|------|------|--------|------|
| F1 | 远程桌面控制 | P0 | 控制端实时查看并操作被控端桌面 |
| F2 | 远程桌面客户端 | P0 | 部署在 Windows 11，支持屏幕捕获、视频编码、输入注入 |
| F3 | 中继服务器 | P0 | 部署在 Ubuntu 云服务器，负责设备发现、会话配对、数据转发 |
| F4 | 连接管理 | P0 | 控制端查看在线设备列表，点击目标设备直接连接，无需被控端同意 |
| F5 | 锁屏下远程 | P0 | 被控端锁屏状态下能查看画面并输入密码解锁 |
| F6 | 断开后锁屏 | P0 | 远程连接断开后，被控端自动进入锁屏状态 |
| F7 | 双屏适配 | P1 | 被控端双屏时，控制端可选择查看任意一个屏幕 |
| F8 | 开机自启 | P0 | 客户端作为 Windows 服务开机自动运行 |

### 非功能需求

| 编号 | 需求 | 优先级 | 说明 |
|------|------|--------|------|
| NF1 | 画质优先 | P0 | 1080P 60fps，H.264 硬件编码，CBR 10-20 Mbps |
| NF2 | 流畅体验 | P0 | 视频走 QUIC datagram（不阻塞），输入走 stream（可靠） |
| NF3 | 低延迟 | P0 | zerolatency 编码参数，QUIC 0-RTT 握手 |
| NF4 | 断线重连 | P2 | 第一版手动重连按钮，不做自动重连 |

### 不做的事情

| 编号  | 内容     | 原因                             |
| --- | ------ | ------------------------------ |
| W1  | 文件传输   | 不在当前范围                         |
| W2  | 声音传输   | 不在当前范围                         |
| W3  | 端到端加密  | 个人自用，QUIC TLS 传输加密已足够；预留接口后续可加 |
| W4  | 自动重连   | 第一版保持简单，手动重连                   |
| W5  | NAT 打洞 | 3-5 台设备云中继转发延迟可接受，架构预留打洞扩展点    |
| W6  | 浏览器客户端 | 仅 Windows 客户端                  |
| W7  | 移动端客户端 | 仅 Windows 客户端                  |

## 用户场景

### 场景 1：日常远程办公

```
你在家（van-laptop）→ 打开客户端 → 看到 van-pc 在线
→ 点击 van-pc → 直接连上 → 操作公司电脑桌面
→ 用完断开 → 公司电脑自动锁屏
```

### 场景 2：锁屏后远程

```
你在家（van-laptop）→ 打开客户端 → 看到 van-pc 在线
→ 点击 van-pc → 连上，看到锁屏画面
→ 输入密码 → 进入桌面 → 正常操作
→ 用完断开 → 自动锁屏
```

### 场景 3：双屏切换

```
被控端有两块屏幕（屏1: VS Code, 屏2: 浏览器）
→ 控制端默认看到屏 1 → 想看浏览器 → 切到屏 2
→ 被控端切换编码源到屏 2 → 控制端画面切换
```

### 场景 4：多设备互连

```
3 台设备都在线：van-pc, van-laptop, van-server
→ 任一打开客户端都能看到另外两台
→ 任一连任一，没有限制
```

## 系统架构

### 整体拓扑

```
┌──────────────┐     QUIC      ┌──────────────┐     QUIC      ┌──────────────┐
│  被控端 A     │◄────────────►│   中继服务器   │◄────────────►│  控制端 B     │
│  (Windows 11) │              │  (Ubuntu)     │              │  (Windows 11) │
│               │              │               │              │               │
│  屏幕捕获      │              │  设备注册表    │              │  视频解码      │
│  H.264 编码   │              │  配对管理      │              │  画面显示      │
│  输入注入      │              │  数据转发      │              │  输入捕获      │
│  锁屏控制      │              │  带宽控制      │              │               │
└──────────────┘              └──────────────┘              └──────────────┘
```

### 连接流程

```
1. 启动阶段
   客户端 A → 中继：Register { device_id: "van-pc", auth_token }
   客户端 B → 中继：Register { device_id: "van-laptop", auth_token }
   中继：维护在线设备表 { "van-pc" → connection_A, "van-laptop" → connection_B }

2. 连接阶段
   客户端 B → 中继：Pair { target: "van-pc" }
   中继：查到 van-pc 在线 → 转发键鼠事件和视频帧
   中继：中继不解密数据，纯转发

3. 断开阶段
   客户端 B → 中继：Disconnect
   中继：通知 van-pc 断开连接
   客户端 A：触发锁屏 → 保持在线，等待下次连接
```

### 数据流

```
被控端：
  DXGI 捕获 D3D11 纹理 → FFmpeg H.264 编码 → QUIC datagram → 中继 → 控制端

控制端：
  中继 → QUIC datagram → FFmpeg 软解 → D3D11 纹理上传 → egui 窗口渲染

输入路径：
  控制端 egui 事件捕获 → 坐标缩放映射 → QUIC stream → 中继 → 被控端
  → SendInput 注入

控制消息：
  双向 QUIC stream（可靠）: Register / Pair / Disconnect / 切屏 / 心跳 / 关键帧请求
```

## 客户端进程架构

```
myowndesk-client.exe
  │
  ├── 服务模式（--service）
  │     ├── Windows 服务注册，开机自启
  │     ├── Session 0 运行，LOCAL SYSTEM 权限
  │     ├── 负责：屏幕捕获、编码、输入注入、锁屏控制、网络连接
  │     └── 本地 TCP 端口监听（供 GUI 连接）
  │
  └── GUI 模式（默认双击启动）
        ├── Session 1 用户桌面运行，普通权限
        ├── 连接后台服务（127.0.0.1 TCP）
        ├── 展示：在线设备列表、连接状态
        └── 操作：发起连接、断开、切屏
```

## 开发阶段

### 第一阶段：核心链路（P0）

目标：单屏远程桌面可用（被控端已登录桌面状态）

1. 搭建三个 crate 骨架、Proto 定义
2. 中继服务器：设备注册 + 配对 + 数据转发
3. 客户端服务模式：DXGI 捕获 + H.264 编码 + QUIC 发送
4. 客户端 GUI 模式：设备列表 + 连接 + H.264 解码 + 画面渲染
5. 输入回传：控制端按键鼠标 → 被控端 SendInput

### 第二阶段：双屏（P1）

6. 被控端多显示器枚举
7. 控制端切屏控制
8. 坐标映射（单屏分辨率不一致 + 双屏）

### 第三阶段：锁屏（P0 难点）

9. 服务模式下桌面切换（Session 0 → 用户桌面）
10. 锁屏安全桌面输入注入
11. 断开后自动锁屏

### 第四阶段：完善

12. 断线重连
13. 端到端加密
14. NAT 打洞（可选）
